# AECOM UI — Design Tokens

The complete token system behind `assets/aecom-theme.css`. Every value below is
already a CSS variable in that file; this doc explains *why* and *how to use them*.

## Color

```
--aecom-teal     #044955   primary dark — nav, hero, dark sections, headers
--aecom-teal-700 #0E5C68   hover / faint backdrop teal
--aecom-green    #00A651   THE accent — one CTA / one highlight per view
--aecom-green-600#009149   green hover
--aecom-turq     #0E8F8C   bridge tone, used sparingly (icons, a second series)
--aecom-ink      #1A1A1A   body text
--aecom-gray     #8C8279   warm gray — captions, muted labels
--aecom-line     #E4E7E7   hairline borders
--aecom-panel    #F4F6F6   off-white card/section fill
--aecom-white    #FFFFFF
```

**Discipline (this is what makes it read as AECOM and not generic):**
- ~60% white, ~30% teal, ~10% green. Never even weight.
- Green is reserved for **one** thing per view: the primary CTA, one stat, one eyebrow. Headers are teal, not green; body is ink, not green. No green backgrounds-as-wash.
- Dark sections use teal `#044955`, **not near-black** — this is the deliberate move that keeps the design off the "acid-green on black" AI-default look. On teal, brighten the green to `#5FE0A8` for contrast.
- Text on teal: white body, brightened-green highlight. Never ink-on-teal.

## Typography

Two roles. Load real faces in `<head>` for the authentic feel; the stacks fall back cleanly with zero dependencies.

- **Display** (`--aecom-font-display`): a geometric sans — Archivo / Futura / Century Gothic. Used for headings, eyebrows, stats, and the wordmark. Bold, slightly tight tracking on large sizes.
- **Body** (`--aecom-font-body`): a neutral humanist sans — Inter / system-ui.
- **Wordmark**: the text `AECOM`, display face, **800 italic, uppercase**, `letter-spacing: 0.06em` (echoes the real logotype's slanted geometric caps). Don't fabricate the official logo SVG; swap in the client's asset if supplied.

Type scale is fluid (`clamp()`), from `--aecom-step--1` (captions) to `--aecom-step-4` (hero display / stat numbers). Don't hand-pick pixel sizes — use the steps so it stays responsive.

Recommended `<head>` (optional, for the real faces):
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Archivo:ital,wght@0,600..800;1,700..800&family=Inter:wght@400;500;600&display=swap" rel="stylesheet">
```

## Spacing, shape, depth

- `--aecom-pad-section` fluid section padding; `--aecom-gap` for grids.
- Radii: `--aecom-radius` (10px cards) / `--aecom-radius-sm` (6px buttons). Restrained, not pill-shaped.
- Shadows are teal-tinted and soft (`--aecom-shadow` / `--aecom-shadow-lg`) — depth without heavy drop shadows.
- `--aecom-maxw` 1200px content column, centered via `.aecom-container`.

## Motion (minimal but present)

Same philosophy as the AECOM deck: a little, done well.
- **Scroll reveal**: `[data-reveal]` fades up 14px when it enters view (via `aecom-reveal.js`). Stagger groups with `data-reveal-group="120"` or per-element `data-reveal-delay`.
- **Hero load**: direct children of `.aecom-hero__inner` rise in, lightly staggered, on load.
- **Micro-interactions**: cards lift 4px on hover; buttons depress 1px on active; nav links fade. Durations 180–480ms on a single soft easing (`--aecom-ease`).
- **Quality floor**: `prefers-reduced-motion` disables all of it; `:focus-visible` shows a green outline. Keep these.

Don't animate everything — a hero sequence plus reveals on section intros is the whole budget. Scattered effects read as AI-generated.

## The signature: building skyline

The recurring, identity-carrying element. Generated by `scripts/make_skyline.js` as
clean SVG (web-native — no licensing, scales infinitely). Two roles:
- **Hero backdrop**: faint (`opacity ~0.2`) skyline pinned to the hero's bottom edge (`.aecom-hero__skyline`).
- **Footer band**: a solid teal skyline strip flush above the footer (`.aecom-skyline-band`).

Regenerate variants (different cities/rhythms) by changing `--seed`; match `--fill` to the surface it sits on.
